## Sequence grammar

A training corpus is one flat token stream laid out like this:

```
corpus         := <CONTEXT_START> <REGION_*> <CLIMATE_*> <DENSITY_*> <CONTEXT_END>
                  object+

object         := <KIND_START> <TAG_KEY_VALUE> extra_attr* <X_#> <Y_#> geometry? <KIND_END>

extra_attr     := <SURFACE_*> | <LIT_*> | <LANES_*> | <ACCESS_*>
                  | <ONEWAY_*> | <BRIDGE_YES> | <TUNNEL_YES>
                  | <SPEED_*> | <LEVELS_*> | <HEIGHT_*>

geometry       := (<dx_N> <dy_M>)+                              (linear feature)
                  | (<dx_N> <dy_M>)+ (<PART_SEP> <X_#> <Y_#> (<dx_N> <dy_M>)+)*   (multipolygon)
                  | (empty)                                     (POI)
```

- `KIND`  ∈ `CONTEXT`, `BUILDING`, `ROAD`, `POI`, `LANDUSE`, `WATERWAY`,
           `RAILWAY`, `NATURAL_LINE`
- `<X_#>` / `<Y_#>` place the object inside a **chunk-local 256×256 grid**.
- `<dx_#>` / `<dy_#>` are **1-meter relative jumps** in the range
  `[-32, +32]`. Longer edges are expressed as multiple (dx, dy) token
  pairs. Vertex N's position is `vertex[N-1] + (dx, dy)` in meters.
- Tags carry the OSM key: `<TAG_AMENITY_HOSPITAL>`, `<TAG_SHOP_SUPERMARKET>`,
  `<TAG_BUILDING_HOUSE>`, etc. Unknown values collapse to
  `<TAG_<KEY>_OTHER>`.
- The three fields `name=*`, `operator=*`, and any other high-cardinality
  identifier tags are **intentionally excluded**.
